# 豆仓 Coffee Vault

本地优先的个人咖啡豆管理 App，支持 Android 与 Web。

默认完全离线：数据保存在设备本地（Android SQLite / Web IndexedDB），不需要账号或网络，拍照识别只申请相机权限。可选登录开启跨设备云同步：登录后数据（含图片）经 HTTPS 传输并保存在云端、仅你自己可见、可随时删除云端账号；不登录则始终不联网、不上传任何数据。

## 功能

### 咖啡豆管理
- 新增、编辑、删除、搜索，按状态筛选与多维度排序（含按单价排序）
- 记录开封后最佳赏味期天数，首页显示赏味倒计时，并提醒赏味将至或余量不足的豆子
- 总豆款、饮用中数量、剩余总克重概览
- 烘焙商 / 产地 / 处理法 Smart Select，支持批量改名和删除
- 每克单价自动计算，单价单位可选每克 / 50g / 100g / 斤
- 可记录购买链接，详情页点击交由系统打开对应 App 或浏览器
- 咖啡袋 / 标签图片：编辑页手动添加，详情展示，点击查看大图并可经系统面板保存或分享

### 拍照识别（离线）
- APK 内置 ML Kit 模型，离线识别中英文包装标签
- 本地解析日期、克重、处理法、烘焙度、产地与风味
- 识别结果进入编辑器等待确认，低可信字段以黄色标出
- 针对表格式标签和 Origin 横排标签增强解析；识别图片可留存为咖啡袋或标签

### 饮用记录与统计
- 「喝一杯」记录用量并扣减余量，记录支持长按删除
- 冲煮参数与方案字段对齐，支持研磨设备、咖啡机、浸泡环境等方式专有字段
- 高级评价多维雷达图（详情与列表同步，每个维度附「?」说明）
- 咖啡日历：月历详情、连续年历格子图、当天花费与评分浮层
- 首页搜索豆名、冲煮方式与备注；首次为某豆记录时自动回填开封日期

### 冲煮方案与辅助
- 按冲煮方式管理方案，「喝一杯」时优先显示上次、已绑定和同方式方案
- 饮用记录中的冲煮参数可另存为新方案；方案编辑可快速选择或新建历史设备记录
- 冲煮辅助按方案步骤自动计时，突出本段注水量、常驻下一段预览、可手动结束并记录实际用时

### 分享
- 咖啡豆、冲煮方案、咖啡日历生成统一票据风分享卡片，经系统面板分享 PNG
- 冲煮方案可生成离线分享码（`DC1-`）并可附二维码，接收方拍照扫码、相册选图或粘贴分享码即可导入为新方案
- 分享卡片可保存到系统相册 `Pictures/豆仓分享卡`

### 云同步（可选）
- 登录后在 Android 与 Web 间同步咖啡豆、饮用记录、冲煮方案和图片
- 默认关闭，不登录不联网；网络请求带超时保护，连不上时明确提示
- 合并采用 last-write-wins + 墓碑软删除，图片内容寻址去重；详见 [docs/SYNC.md](docs/SYNC.md)

### 界面与体验
- 深烘、澳白、火山灰绿、生豆四套主题
- 新用户空豆仓引导、「喝一杯」发现提示、进阶功能轻提示
- 右下浮动按钮空闲后自动收起，减少遮挡列表
- 全面屏状态栏、安全区域与小数水量输入针对 Android 实机优化
- Android 返回键分层关闭弹窗，应用恢复时刷新数据库
- 自适应图标；Android 12+ 系统启动画面使用矢量豆图标与手写 slogan 品牌条，冷启动无插件停留
- Web 版为 PWA，Service Worker 按版本整套预缓存，联网秒开、离线可用

## 安装

推荐安装正式签名版：`dist/coffee-vault-2.2.1-release.apk`

把 APK 发送到 Android 手机，打开并按系统提示允许本次「安装未知应用」。已安装旧版时，只要沿用同一发布密钥并提高 `versionCode`，即可覆盖升级并保留数据库。

> 默认只发布正式签名版。调试包仅用于开发验收，与正式版签名不同，不能直接覆盖正式版。

## 数据安全

- 正式数据位于应用私有 SQLite 数据库；卸载应用会删除本机数据库，请先导出备份。
- 设置页可按范围导出（全部 / 豆仓与饮用记录 / 冲煮方案），并可事务式恢复备份。
- 导入可选合并或覆盖：合并时同 ID 数据保留更新时间较新的版本；导入失败会回滚，不覆盖当前数据。
- 完整或豆仓备份可选包含咖啡袋、标签与饮用记录照片，恢复时写回应用私有图片目录。
- `release-keystore/` 保存后续升级所需的发布密钥与密码，已被 `.gitignore` 排除；请把整个目录另行备份并保持私密。

## 项目结构

- `www/`：界面、交互与数据适配层（`data-core.js` 纯逻辑，`repository.js` 存储与迁移，`sync-*.js` 云同步）
- `worker/`：Cloudflare Worker 同步后端（D1 + R2）
- `android/`：Capacitor Android 工程
- `tests/`：数据规范化、筛选排序、备份与同步逻辑测试
- `resources/`、`assets/`：图标与启动画面源文件
- `docs/BUILDING.md`：构建、签名与版本升级说明
- `docs/CHANGELOG.md`：版本更新记录
- `docs/SYNC.md`：云同步架构（LWW 合并、增量游标、图片内容寻址、后端接口）
- `AGENTS.md`：项目约定与协作规范（含分享卡片的内容 / 渲染分离等架构约定）

分享卡片遵循内容与渲染分离：`data-core.js` 的 `buildSharePayload` 只整理分享内容，`app.js` 的 Canvas 渲染器（经 `SHARE_CARD_RENDERERS` 注册）负责绘制与系统分享。

## 技术信息

- 包名 `com.coffeebean.vault`，版本 `2.2.1`（versionCode `49`）
- 最低 Android 7 / API 24，目标 Android 16 / API 36
- 前端原生 HTML/CSS/JS，无构建步骤；Android 壳为 Capacitor 8
