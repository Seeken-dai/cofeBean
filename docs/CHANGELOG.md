# 更新记录

本项目采用 [Keep a Changelog](https://keepachangelog.com/zh-CN/1.1.0/) 格式，并遵循语义化版本。

## [2.0.0] - 2026-07-02

### 新增

- Web 端新增可选云同步：登录后同步咖啡豆、饮用记录、冲煮方案与图片；不登录时仍保持默认离线、零网络。
- 新增同步账号注册、登录、恢复码、立即同步、前台自动同步、退出账号与删除云端账号。
- 新增 Cloudflare Worker + D1 + R2 同步后端，支持 LWW + 墓碑软删除、图片内容寻址去重和用户隔离。

### 变更

- 项目定位更新为“本地优先 + 可选同步”；1.x 本地优先线保留在 `release/1.x-local-first` 分支。
- Android 2.0.0 同步版新增 `INTERNET` 权限用于可选云同步。

### 修复

- 修复 Web 创建并同步到 Android 的咖啡豆在 Android 端无法删除的问题；原生删除路径不再引用冲煮方案表不存在的同步列，也不依赖 SQLite JSON 扩展。

### 发布

- Android 版本号更新为 `versionName 2.0.0`、`versionCode 35`；正式签名版 `dist/coffee-vault-2.0.0-release.apk`。
- 构建前通过 `node --check www/app.js`、`node --check www/repository.js`、`node --check www/sync-service.js` 和 `npm.cmd test`（83/83）。

## [1.5.2] - 2026-07-01

### 改进

- 咖啡袋与标签图片归档时自动压缩为 WebP（长边最长 1600 像素、质量 80），显著降低本机占用与备份体积，标签文字仍清晰可读。
- 压缩仅在图片归档时进行，不影响拍照识别（OCR 仍使用原始高清图）；压缩失败时回退为原样保存，不丢图。

### 发布

- Android 版本更新为 `versionName 1.5.2`、`versionCode 34`，正式签名版 `dist/coffee-vault-1.5.2-release.apk` 待构建验证。

## [1.5.0] - 2026-06-30

### 新增

- 冲煮方案新增离线分享码 `DC1-`，可完整携带单个方案的名称、冲煮方式、关键参数和分段步骤。
- 冲煮方案分享卡片可选择附带二维码，对方用豆仓扫码即可导入方案。
- 新增“导入分享方案”入口，支持拍照扫码、相册选图和粘贴分享码三种方式。

### 改进

- 分享码带短校验位，截断、篡改、非豆仓分享码会给出明确错误，不静默生成错误方案。

- 冲煮方案功能关闭时仍可从设置入口导入分享方案，导入成功后自动开启方案页。
- 导入弹窗使用内联状态提示，避免扫码反馈被系统弹窗层级遮挡；拍照扫码增加缩图和反色重试以提升识别率。

### 发布

- Android 版本更新为 `versionName 1.5.0`、`versionCode 32`，已生成正式签名版 `dist/coffee-vault-1.5.0-release.apk`。

## [1.4.8] - 2026-06-30

### 新增

- 新增咖啡豆只读详情页，并提供独立的编辑图标入口。
- 新增开封日期字段，旧购买日期数据继续保留以保证兼容。

### 变更

- 收紧豆仓卡片高度、间距和信息层级，提高列表空间利用率。
- “快捷喝一杯”改为带克数角标的杯形图标按钮，不再单独占用一行。

## [1.2.0] - 2026-06-24

### 新增

- 独立饮用记录系统，支持用豆克数、冲煮方式、时间、整体评分、备注与七维评价。
- “豆仓 / 饮用”双标签首页、全局饮用时间线和单款咖啡豆饮用历史。
- 卡片“喝一杯”快捷入口，可在设置中调整默认克数。
- 高级评价开关与香气、酸质、甜感、醇厚度、余韵、平衡度、苦涩感显示控制。
- 关于页，展示真实版本、产品介绍、最新功能与离线隐私说明。

### 变更

- SQLite 数据库升级至 schema v2；饮用记录的新增、修改与删除会在事务中同步调整库存。
- JSON 备份升级至 schema v2，包含豆子、饮用记录和偏好设置，并兼容 v1 备份。
- 删除咖啡豆后保留其饮用历史与豆名快照。

## [1.1.1] - 2026-06-22

### 修复

- 修正部分三星设备上拍照与从相册读取按钮文案相反的问题。

## [1.1.0] - 2026-06-21

- 识别结果置信提示与保存前人工确认。
