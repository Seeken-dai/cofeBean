# 更新记录

本项目采用 [Keep a Changelog](https://keepachangelog.com/zh-CN/1.1.0/) 格式，并遵循语义化版本。

## [2.3.1] - 2026-07-12

### Fixed

- 多层弹窗改用单一全局遮罩，修复关闭二级、三级及更深层弹窗后 Android WebView 短暂整屏发黑的问题；数字滚轮等独立打开的弹层也统一纳入遮罩管理。
- 冲煮计时器移除难以稳定对齐的水滴端点，重做为独立可见的水波呼吸层，并在暂停时冻结动画。
- 冲煮时钟改为只给变化的数字位播放切换动画，避免每秒整个时钟一起跳动。
- 「喝一杯」入口的勾选反馈在叉号、取消、返回键或遮罩关闭弹层后都能恢复克数。
- 冲煮完成触感改用 Capacitor 原生 Haptics，并保留浏览器振动降级。

### Changed

- Android 版本更新为 `versionName 2.3.1`、`versionCode 59`；原生 Haptics 会加入振动权限，不新增网络、相册、存储或定位权限。

## [2.3.0] - 2026-07-12

### Added

- 豆卡、饮用记录与冲煮方案统一使用带轻微回弹的分层入场动效，余量条随卡片展开；详情高级评价雷达图新增网格、数据形状、顶点与波纹的分段生长动画。
- 「喝一杯」入口新增杯子倾斜、勾选和克数浮字反馈，顶部豆款、饮用中与剩余总量支持数值滚动更新。
- 冲煮辅助圆环新增发光水滴端点、注水阶段水波呼吸、阶段数字切换动画与完成触感反馈。

### Changed

- 全部弹窗统一为弹簧入场和下滑退场；豆仓、饮用记录与方案 Tab 切换增加左右方向感。

- Android 版本更新为 `versionName 2.3.0`、`versionCode 58`。

## [2.1.2] - 2026-07-05

### 改进

- 退出应用改为轻提示：主界面再次返回（约 2 秒内）才退出，先 toast 提示「再次操作退出程序」，移除原来的整块退出确认弹窗。
- 高级评价每个维度新增「?」说明浮窗（点击显示，Popover API 轻触关闭，不支持时回退 toast），告知该维度含义与打分方式；饮用记录编辑与设置的评价维度配置处均有。

### 修复

- 冲煮辅助圆环在段末/间奏开始处的跳切：`brewAssistStatus` 原本把 elapsed 取整判定段落，而圆环进度按原始秒计算，临界点相差约 0.5 秒导致圆环快满时被提前截断。改为用原始秒判定段落，仅显示用的 elapsed 取整。
- 每次发新版后 Web 打开报错：原 Service Worker 用 stale-while-revalidate 且缓存名固定，各外壳文件独立更新，弱网下会出现新旧文件错配（如新 index.html 配旧 app.js）。改为按版本原子化预缓存（`addAll` 全成功才启用、激活时清旧缓存、cache-first），缓存名随版本变化（由 `bump-version.mjs` 同步），并移除「非导航请求回退 index.html」以免把 HTML 当脚本返回。
- 同步服务器改用自定义域名（Cloudflare Worker Custom Domain，绑定同一 Worker 与同一份数据），缓解 `*.workers.dev` 泛域在部分国内网络下被 DNS 污染导致的登录/同步连不上。仅改动客户端 `www/sync-transport.js` 的 `DEFAULT_BASE_URL`；旧地址仍指向同一后端，已发布旧版本继续可用，无数据迁移。（版本号不变，重新出包生效。）
- 修复 Web 版整站打不开（Safari 报 `Response served by service worker has redirections`、Chromium 报 `ERR_FAILED`）：Service Worker 预缓存了 `./index.html`，而 Cloudflare Pages 会把 `/index.html` 308 重定向到 `/`，缓存到的响应带 `redirected` 标记，导航返还时被浏览器拒绝。改为导航走根路径 `./`（直接 200）、不再预缓存 `./index.html`，并对返还响应做 `redirected` 剥离兜底；`CACHE` 名加 `-hotfix1` 强制清除旧的被污染缓存。仅影响 Cloudflare Pages 托管的 Web 版，Android App（本地 `capacitor://` 加载）不受影响。

### 发布

- Android 版本保持 `versionName 2.1.2`、`versionCode 45`；正式签名版 `dist/coffee-vault-2.1.2-release.apk`（含同步自定义域名修复，重新出包覆盖）。
- 构建前通过 `node --check www/app.js`、`node --check www/data-core.js`、`node --check www/repository.js`、`node --check www/sw.js`、`node --check www/sync-transport.js` 与 `npm.cmd test`（117/117）。
